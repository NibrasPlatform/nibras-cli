# CS106L (Standard C++ Programming)

This repo now vendors the CS106L assignment starters under `courses/cs106l/assignments/`
so the course is self-contained and does not depend on the original Downloads folder.

## Project IDs

Use these IDs with the legacy CLI:

| ID          | Folder                                      |
| ----------- | ------------------------------------------- |
| `gapbuffer` | `courses/cs106l/assignments/GapBuffer`      |
| `hashmap`   | `courses/cs106l/assignments/HashMap`        |
| `kdtree`    | `courses/cs106l/assignments/KDTree`         |

## Checks

Each project uses command-based checks:

- Requires `cmake`, `ctest`, and a C++ compiler on the local machine.
- Runs `cmake -S . -B build`, then `cmake --build build`, then `ctest --test-dir build --output-on-failure`.
- Uses the assignment folder itself as the working directory.

If `cmake` is missing, `nibras` exits with a clear prerequisite error instead of silently falling back.

## Setup

`nibras <subject> setup <project>` copies the vendored starter directory into a new local folder:

- `gapbuffer` → `nibras-cs106l-gapbuffer`
- `hashmap` → `nibras-cs106l-hashmap`
- `kdtree` → `nibras-cs106l-kdtree`

Override the destination with `--dir` when needed.

## Commands

```bash
nibras cs106l task gapbuffer
nibras cs106l setup gapbuffer
nibras cs106l test gapbuffer

nibras cs106l task hashmap
nibras cs106l test hashmap

nibras cs106l task kdtree
nibras cs106l test kdtree
```
