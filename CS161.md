categories: `Exams/` and `sections/`. Each item has its own folder and a
`scores.json` file for grading.

## Grading

Each project uses `check` grading. For instructor-only grading, keep
`grading.json` in a private grading repo and set `NIBRAS_GRADING_ROOT` (or pass
`--grading-root`). `nibras` will auto-check answers using one file per question
(e.g., `q1.txt`). If no grading file is found, it falls back to `scores.json`
(manual percentage). You can store answers in a different folder by passing
`--answers-dir`.

`score = earnedPoints / totalPoints * 100`

Auto-check example (`grading.json`, strict exact match):

```json
{
  "totalPoints": 100,
  "questions": [
    {
      "id": "q1",
      "points": 45,
      "answerFile": "q1.txt",
      "solutions": ["Answer A", "Answer B"]
    }
  ]
}
```

Auto-check rules:
- Trim and collapse whitespace.
- Case-sensitive exact match.
- No partial credit.

Manual grading (`scores.json`):

```json
{
  "earnedPoints": 60,
  "totalPoints": 100
}
```

You can omit `totalPoints` if it is already set in `.nibras.json`.
Validation rules:
- `grading.json` must exist when `NIBRAS_GRADING_ROOT` or `--grading-root` is set.
- Question IDs must be unique.
- Sum of question points must equal `totalPoints`.
- Each `answerFile` must exist and be non-empty.
- `earnedPoints` must be >= 0
- `totalPoints` must be > 0
- `earnedPoints` must be <= `totalPoints`

## Commands

```bash
nibras cs161 test exam1
nibras cs161 test exam1 --answers-dir /path/to/my/answers
nibras cs161 test exam1 --grading-root /path/to/private/grading
nibras cs161 test section4
nibras cs161 submit exam2

nibras cs161 setup exam1
```
